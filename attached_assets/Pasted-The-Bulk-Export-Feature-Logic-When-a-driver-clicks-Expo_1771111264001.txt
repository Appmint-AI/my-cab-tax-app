The "Bulk Export" Feature Logic

When a driver clicks "Export," the app needs to do three things in the background:

Filter: Grab all records for the selected tax year (e.g., 2026).

Compile: Generate a professional PDF report and a raw CSV (spreadsheet) file.

Zip: Bundle all the receipt images into a single folder so the user has the "Receipt Vault" on their local hard drive as a backup.

📝 Instructions for the Replit Agent

Copy and paste this into your Replit Agent to build the export engine:

"I need to build a 'Tax Ready Export' feature for the Web Dashboard.

Libraries: Use jspdf and jspdf-autotable for PDF generation and json2csv for the spreadsheet export.

The Report: Create a 'Year-End Summary' PDF that includes:

A total breakdown of Gross Income vs. Total Deductions.

A categorized list of expenses (Fuel, Maintenance, Equipment, etc.).

A summary of Total Mileage and the calculated deduction ($0.725/mile for 2026).

The Vault Backup: Create a button that allows the user to download a .zip file containing all their uploaded receipt images, organized by month.

Desktop UI: Place these buttons prominently on a new page called /export that is only visible in the Sidebar of the Desktop/Web view."
